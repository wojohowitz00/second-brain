# Phase 10.2: Swift Migration - Menu Bar UI

**Goal**: Implement the native macOS Menu Bar application interface and integrate the backend services.

## User Review Required

> [!NOTE]
> This phase transforms the command-line executable into a GUI application that lives in the Menu Bar.

## Proposed Changes

### App Entry Point
#### [NEW] [apple/SecondBrain/Sources/SecondBrain/SecondBrainApp.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/SecondBrainApp.swift)
- **Attribute**: `@main`
- **Responsibility**: Bootstraps `NSApplication` and sets the delegate.

### Menu Bar Logic
#### [NEW] [apple/SecondBrain/Sources/SecondBrain/UI/MenuBarManager.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/UI/MenuBarManager.swift)
- **Class**: `MenuBarManager`
- **Responsibility**: Manages `NSStatusItem`.
- **Features**:
    - Sets icon (SF Symbol: `brain.head.profile` or similar).
    - Builds the `NSMenu`.
    - Handles updates (e.g., changes status dot based on Ollama connection).

### Integration
- **Wire up**: `OllamaClient` checks status on launch.
- **Menu Items**:
    - "Status: Checking..." (Dynamic)
    - "Sync Now" (Placeholder action)
    - "Quit"

## Verification Plan

### Manual Verification
*   **Run**: `swift run`
*   **Check**:
    *   Menu bar icon appears.
    *   Clicking icon reveals menu.
    *   "Status" updates to "Connected" (if Ollama is running) or "Disconnected".
    *   "Quit" terminates the app.
