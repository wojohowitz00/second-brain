# Phase 10.4: Swift Migration - Core Processing

**Goal**: Implement the core application logic: fetching messages from Slack, classifying them via Ollama, and saving them as Markdown files in the Vault.

## User Review Required

> [!CAUTION]
> This phase implements the actual "work" of the application. It requires a valid Slack API Token (which should be added to Settings).

## Proposed Changes

### 1. Slack Integration
#### [NEW] [apple/SecondBrain/Sources/SecondBrain/Services/SlackClient.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/Services/SlackClient.swift)
- **Class**: `SlackClient`
- **Methods**: `fetchHistory(channelID: String, latest: String?) async throws -> [SlackMessage]`.
- **Logic**: Uses `URLSession` to call `users.conversations.history` (or `conversations.history`).
- **Dependencies**: Needs API Token (from Settings/Keychain).

### 2. Processing Logic
#### [NEW] [apple/SecondBrain/Sources/SecondBrain/Services/NoteProcessor.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/Services/NoteProcessor.swift)
- **Class**: `NoteProcessor`
- **Responsibility**: Orchestrates the pipeline.
- **Flow**:
    1.  Fetch unread messages from Slack.
    2.  For each message:
        *   Call `OllamaClient.classify(message)` (Need to implement classification prompt).
        *   Call `VaultScanner` to resolve path.
        *   Write `.md` file to disk.

### 3. Settings Enhancement
- **Update**: `SettingsView` / `SettingsViewModel` to include **Slack API Token** and **Channel ID** fields.

### 4. Integration
- **Wire up**: Connect `MenuBarManager.syncNow()` to `NoteProcessor.processInbox()`.
