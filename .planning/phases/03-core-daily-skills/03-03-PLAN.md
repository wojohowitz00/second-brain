Replace the content of `.claude/commands/new-task.md` with the enhanced version. This is a command instruction file that tells Claude how to process `/new-task` invocations.

The new command must specify:

**Two modes:**

1. **Standard mode** (`/new-task [description]`): Parse natural language, extract all fields, create task, confirm, offer to add notes.
2. **Quick mode** (`/new-task quick [title]`): Minimal — title + status: backlog + created date. No prompts. No inference.

**Standard mode workflow:**

Step 1 — Parse the description and extract:

- **title**: Clean, concise task title (sentence case, strip filler words like "I need to")
- **due_date**: Bare ISO date (no quotes). Inference rules:
  - "by Friday" → next Friday's date
  - "end of week" → coming Sunday
  - "tomorrow" → tomorrow's date
  - "EOD" / "end of day" → today's date
  - "next week" → next Monday
  - "end of month" → last day of current month
  - If no date mentioned → omit field entirely (not null, omit)
- **status**: Default `backlog` (user can override, but default is backlog — not active)
- **priority**: Enum `high | medium | low`. Inference:
  - "urgent", "critical", "ASAP", "important", "high priority" → `high`
  - "someday", "eventually", "when I have time", "low priority" → `low`
  - Default → `medium`
- **project**: Wikilink format `"[[ProjectName]]"` (quoted in YAML). Inference:
  - If description mentions a known project name (check `projects/` folder), link it
  - If no project mentioned → omit field entirely
- **domain**: Enum `sales | content | product | admin | research | people`. Inference:
  - "email", "call", "schedule", "invoice", "admin" → `admin`
  - "write", "post", "article", "video", "content" → `content`
  - "research", "read", "learn", "study", "explore" → `research`
  - "sales", "client", "deal", "pipeline", "prospect" → `sales`
  - "build", "code", "ship", "feature", "bug" → `product`
  - Person names, "follow up with", "check in with" → `people`
  - Default → omit field (don't guess if no signals)
- **context**: Enum `quick | deep | collab`. Inference:
  - "quick", "5 min", "2 min", "takes a second" → `quick`
  - "review", "analyze", "write", "think through", "deep dive" → `deep`
  - "with [person]", "schedule meeting", "discuss" → `collab`
  - Default → omit field
- **tags**: Array. Include domain tag (e.g., `#admin`) and context tag (e.g., `#quick`) if inferred. Add any explicit tags from the description.
- **created**: Today's date, bare ISO

Step 2 — Create the file:
- Filename: kebab-case from title, no special characters, concise
- Location: `tasks/` directory
- Frontmatter template:

```yaml
---
type: task
title: [extracted title]
due_date: YYYY-MM-DD
status: backlog
priority: medium
project: "[[ProjectName]]"
domain: admin
context: deep
tags: [#admin, #deep]
created: YYYY-MM-DD
---
```

Only include fields that have values. Omit fields with no signal (don't write `domain:` with an empty value — just leave it out).

CRITICAL: `due_date` and `created` must be bare ISO dates. NEVER quote them. Quoted dates break Dataview.

Step 3 — Confirm:
Show the user what was created:
```
Created: tasks/[filename].md
  Title: [title]
  Due: [date or "none"]
  Priority: [priority]
  Status: backlog
  [other fields if present]
```

Step 4 — Offer to add notes (ONE prompt only):
"Want to add any notes or context?"
- If yes: append user's notes below the frontmatter
- If no: done
- Do NOT prompt again after this. One offer, one response, done.

**Quick mode workflow (`/new-task quick [title]`):**

1. Take the title as-is (no inference, no parsing)
2. Create file with minimal frontmatter:
```yaml
---
type: task
title: [title as given]
status: backlog
created: YYYY-MM-DD
---
```
3. Confirm with one line: `Created: tasks/[filename].md — backlog`
4. No prompts. No offers. Done.

**Examples:**

`/new-task Send thank you note to Claire by Friday`
→ title: "Send thank you note to Claire", due_date: [next Friday], priority: medium, domain: people, context: quick, tags: [#people, #quick]

`/new-task Review Q3 sales numbers`
→ title: "Review Q3 sales numbers", priority: medium, domain: sales, context: deep, tags: [#sales, #deep]

`/new-task ASAP fix the login bug`
→ title: "Fix the login bug", priority: high, domain: product, tags: [#product]

`/new-task quick Buy groceries`
→ title: "Buy groceries", status: backlog, created: today. Done.

**Filename convention:**
- kebab-case
- Strip filler ("I need to", "ASAP", "by Friday")
- Keep concise but descriptive
- No special characters
