Extend the existing `session-end.sh` to append a "## Day Summary" section to the daily-brief note for today. The script must preserve the existing dashboard refresh behavior.

The updated script should:

1. **Keep existing behavior:** Refresh `refreshed_by` date on the tasks-by-status dashboard (already implemented).

2. **Add new behavior:** Append a Day Summary section to today's daily-brief note if it exists.

Target file: `$VAULT/05_AI_Workspace/daily-briefs/$(date +%Y-%m-%d)-daily-brief.md`

If the daily-brief file exists AND does not already contain "## Day Summary":
- Append the following:

```
\n## Day Summary\n\n*Session closed: YYYY-MM-DD HH:MM*\n
```

If the daily-brief file does not exist, skip silently (no error). The morning briefing may not have run today.

If "## Day Summary" already exists in the file, skip the append (idempotent — don't add duplicate Day Summary sections from multiple sessions in one day).

3. **Also refresh projects-health dashboard** (will be created by Plan 03-04):
Target: `$VAULT/05_AI_Workspace/dashboards/projects-health.md`
Same `sed` pattern as tasks-by-status: update `refreshed_by` field if file exists.

4. **Performance:** The entire script must complete well within the 5000ms timeout. All operations are simple file checks and appends — expected <200ms total.

5. **Always exit 0** — hook must never fail the session close.

Updated script structure:

```bash
#!/usr/bin/env bash
# session-end.sh — Day Summary append + dashboard refresh at session close
# Runs via SessionEnd hook. Must be fast (well within 5s timeout).
# Always exits 0.

VAULT="/Users/richardyu/Library/Mobile Documents/iCloud~md~obsidian/Documents/Home"
TODAY=$(date +%Y-%m-%d)
NOW=$(date "+%Y-%m-%d %H:%M")

# --- Day Summary append ---
BRIEF="$VAULT/05_AI_Workspace/daily-briefs/${TODAY}-daily-brief.md"
if [[ -f "$BRIEF" ]] && ! grep -q "## Day Summary" "$BRIEF" 2>/dev/null; then
  cat >> "$BRIEF" << EOF

## Day Summary

*Session closed: ${NOW}*

EOF
fi

# --- Dashboard refresh ---
TASK_DASH="$VAULT/05_AI_Workspace/dashboards/tasks-by-status.md"
if [[ -f "$TASK_DASH" ]]; then
  sed -i '' "s/^refreshed_by: .*/refreshed_by: $TODAY/" "$TASK_DASH" 2>/dev/null
fi

PROJECT_DASH="$VAULT/05_AI_Workspace/dashboards/projects-health.md"
if [[ -f "$PROJECT_DASH" ]]; then
  sed -i '' "s/^refreshed_by: .*/refreshed_by: $TODAY/" "$PROJECT_DASH" 2>/dev/null
fi

exit 0
```

IMPORTANT: Use the exact vault path. The spaces in "Mobile Documents" and "iCloud~md~obsidian" are intentional — always quote the path variables.

This skill is the interactive complement to the automatic session-end.sh hook. The hook handles the mechanical write (Day Summary timestamp). This skill handles the reflection question and MEMORY.md capture.

The skill.md must specify:

**Description:**
End-of-day update skill. Captures session learnings and patterns into Claude Code memory for cross-session continuity. Invoked manually before closing a session, or when Claude detects a substantial session is ending.

**Triggers:**
- "end of day"
- "EOD update"
- "wrap up"
- "session learnings"

**Workflow:**

1. **Auto-detect session activity:**
   - List task files modified today (check file modification times in `tasks/`)
   - Note any new files created in `tasks/`, `projects/`, `people/`
   - Summarize: "This session, we [did X, Y, Z]"

2. **Ask ONE reflection question:**
   - "Any decisions or patterns from today's session I should remember for next time?"
   - This is optional — if user says "no" or "skip", proceed without reflection input
   - Keep it to ONE question, not a multi-turn dialog

3. **Capture to MEMORY.md:**
   Before writing, check MEMORY.md line count:
   ```bash
   MEMORY="$HOME/.claude/projects/-Users-richardyu-PARA-01-Projects-Personal-apps-second-brain/memory/MEMORY.md"
   wc -l < "$MEMORY"
   ```
   - If >170 lines: warn user "MEMORY.md is at N lines (target: <150). Consider consolidating before adding more."
   - If <=170 lines: append learnings

   What to capture (append to appropriate section):
   - **Decisions/rationale** → "## Patterns Learned" section (e.g., "User prefers X over Y because Z")
   - **New preferences** → "## User Preferences" section (e.g., "User wants task titles in sentence case")
   - **System facts** → "## System Architecture" or "## Conventions" section if new convention discovered

   Append format: `- [learning] (discovered YYYY-MM-DD)`

   IMPORTANT: Do NOT overwrite MEMORY.md. Read it first, then use Edit tool to append to the appropriate section. Preserve all existing content.

4. **Update daily-brief Day Summary (if exists):**
   If `05_AI_Workspace/daily-briefs/YYYY-MM-DD-daily-brief.md` exists and has a "## Day Summary" section, append session learnings under it:
   ```markdown
   ### Session Learnings
   - [learning 1]
   - [learning 2]
   ```
   If the Day Summary section doesn't exist yet (hook hasn't fired), skip this step — the hook will create it at session close.

5. **Confirm and close:**
   - "Session learnings captured. MEMORY.md at N/200 lines."
   - If no learnings: "No new learnings to capture. Have a good one!"

**Anti-patterns:**
- Do NOT ask multiple reflection questions (one only)
- Do NOT overwrite MEMORY.md (append to sections)
- Do NOT create the Day Summary section (that's the hook's job)
- Do NOT prompt if user says "skip" or "no"
- Do NOT write to vault PARA folders (01-04)
