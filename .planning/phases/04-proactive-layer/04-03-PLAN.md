  <action>
Create the canvas generation skill document at `.claude/skills/surfacing/canvas-review/skill.md`.

This is a Claude Code skill file (instruction document). It tells Claude how to generate a JSON Canvas v1.0 file for the weekly visual review.

The skill must define:

**Data Collection:**
1. Scan `projects/*.md` (skip README.md) — extract `status`, `name`, `health` from frontmatter
2. Scan `tasks/*.md` (skip README.md) — extract `status`, `project`, `title`, `due_date` from frontmatter
3. Group projects by status: active, waiting, blocked
4. For each project, collect ONLY blocked/overdue tasks (problem-focused per CONTEXT.md — not full task list)
5. Cap at 10 projects per lane; if more, add an overflow text node "... and N more"

**Canvas Layout (JSON Canvas v1.0):**

The output file is `05_AI_Workspace/canvas/weekly-review.canvas`.

File must be valid JSON following the JSON Canvas v1.0 spec (https://jsoncanvas.org):

```json
{
  "nodes": [...],
  "edges": [...]
}
```

**Node types used:**
- `group` nodes for swimlanes (Active, Waiting, Blocked)
- `text` nodes for project cards inside each lane

**Swimlane structure:**

Three group nodes arranged horizontally:

| Lane | X position | Color | Label |
|------|-----------|-------|-------|
| Active | -700 | "4" (green) | "Active" |
| Waiting | 0 | "3" (yellow) | "Waiting" |
| Blocked | 700 | "1" (red) | "Blocked" |

Each group: `width: 600`, `height: calculated` (base 200 + 140 per project card).

**Project card format (text nodes):**

Each project is a text node inside its status group:
```
**[[Project Name]]**
Health: good/at-risk/blocked
[If blocked/overdue tasks exist:]
- Task Name (due YYYY-MM-DD) — OVERDUE
- Task Name — BLOCKED
```

Text node dimensions: `width: 560, height: 120` (or taller if many tasks).
Position: stacked vertically inside the group, with 20px padding.

**ID generation:**
Generate unique IDs for each node. Use a deterministic scheme: `lane-{status}` for groups, `proj-{filename-without-ext}` for project cards. This makes the canvas diffable across weeks.

**Edges:** Empty array `[]` — no connections between cards (spatial layout only).

**Regeneration:** The canvas file is overwritten on each `/weekly` run. It is a snapshot, not an evolving document.

**No "Wins" section** — per CONTEXT.md, the board shows active/blocked work only.

**Triggers:**
- Part of `/weekly` workflow (not standalone)
- "canvas review"
- "visual review"

**Error handling:**
- No projects found: generate a minimal canvas with a single text node "No active projects this week"
- Malformed project frontmatter: skip that project, continue
- Canvas folder missing: create it (but it should exist from Phase 1)

**Anti-patterns to document:**
- Do NOT use `.md` extension (must be `.canvas`)
- Do NOT include completed/archived projects
- Do NOT include all tasks (only blocked/overdue — problem-focused)
- Do NOT add edges (spatial layout, not a graph)
- Do NOT include a "Wins" or "Completed" lane
