  <action>
Create the insights detection skill document at `.claude/skills/surfacing/insights/skill.md`.

This is a Claude Code skill file (instruction document, not executable code). It tells Claude how to perform vault-wide pattern analysis when invoked as part of `/weekly`.

The skill must define these detection rules:

**1. Dormant Projects** (PROACT-01)
- Scan `projects/*.md` (skip README.md)
- Extract `status` from frontmatter
- For projects where `status: active`, check file mtime using `stat -f "%Sm" -t "%s" "$file"` on macOS
- Threshold: 14 days of no modification = dormant
- Output: "[[project-name]] dormant for N days — consider archiving or adding a next task"

**2. Neglected Areas** (PROACT-01)
- Scan `tasks/*.md` for domain tags in frontmatter (sales, content, product, admin, research, people)
- Compare domain activity: if a domain has zero tasks modified in 14+ days while other domains have activity, flag it
- Output: "#domain has had no activity in N days while #other-domain is active"

**3. Overcommitment** (PROACT-01)
- Count tasks in `tasks/*.md` where `status: active`
- Threshold: 10+ active tasks triggers warning
- Output: "N active tasks — consider deferring or completing some before adding more"

**4. Goal Drift** (PROACT-01)
- Scan `tasks/*.md` where `status: active`
- Check `project` field in frontmatter
- If `project` field is non-empty but the referenced project file either does not exist or has `status` != active, flag it
- Tasks with empty/missing `project` field: skip silently (per CONTEXT.md decision)
- Output: "[[task-name]] references project [[project-name]] which is inactive/missing"

**Output format** (per CONTEXT.md):
- File: `05_AI_Workspace/insights/YYYY-MM-DD-insights.md`
- Frontmatter: `type: insights`, `date: YYYY-MM-DD` (bare ISO, no quotes), `generated_by: claude`
- Sections: ## Goal Drift, ## Neglected Areas, ## Overcommitment, ## Dormant Projects
- Each insight: observation + recommendation with [[wikilinks]] to specific notes
- Hard cap: 5-10 items total. If more than 10 insights found, prioritize: overcommitment first, then goal drift, then dormant projects, then neglected areas
- Omit empty sections entirely

**Triggers:**
- Part of `/weekly` workflow (not standalone)
- "vault insights"
- "pattern detection"

**Error handling:**
- Missing directories: skip that detection rule, continue with others
- Malformed frontmatter: skip that file, continue
- No insights found: write a brief "No significant patterns detected this week" file

**Anti-patterns to document:**
- Do NOT run on session start (per CONTEXT.md — runs as part of /weekly only)
- Do NOT use Dataview queries (this is a Claude analysis skill, not a vault query)
- Do NOT quote dates in frontmatter
