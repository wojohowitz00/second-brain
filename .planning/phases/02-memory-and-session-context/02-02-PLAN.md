<task type="auto">
  <name>Task 1: Create SessionStart hook script</name>
  <files>.claude/hooks/session-start.sh</files>
  <action>
Create `.claude/hooks/session-start.sh` with the following behavior:

1. Scan all `.md` files in the tasks directory (skip README.md)
2. For each task file, extract `status`, `title`, and `due_date` from YAML frontmatter using grep/awk (NO pyyaml)
3. Skip tasks with `status: done`
4. Categorize into three buckets:
   - **Overdue:** `due_date` is before today (compare strings, ISO format sorts correctly)
   - **Upcoming:** `due_date` is between today and 7 days from now
   - **Stale:** No due_date, but file last modified 7+ days ago AND status is active, waiting, or blocked
5. Output a plain-text summary capped at ~30 lines (max 5 items per bucket)
6. Exit 0 always (even if no tasks found -- just output nothing)

Key implementation details:
- Use macOS `date -v` syntax for date arithmetic (with GNU `date -d` fallback)
- Use `stat -f "%Sm" -t "%Y-%m-%d"` for file modification date on macOS
- Handle edge case: tasks dir has only README.md or is empty -- output nothing, exit 0
- Handle edge case: task file has no frontmatter -- skip it
- Make the script executable: `chmod +x .claude/hooks/session-start.sh`

The script's stdout is injected directly into Claude's session context. Format output as:

```
=== TASK STATUS (YYYY-MM-DD) ===

OVERDUE:
  - Task title (due YYYY-MM-DD, status: xxx)

UPCOMING (next 7 days):
  - Task title (due YYYY-MM-DD)

STALE (no activity 7+ days):
  - Task title (last modified YYYY-MM-DD, status: xxx)

===================================
```

Only print sections that have items. If no items in any category, print nothing (silent exit 0).
