## docs/specs/ — Design Intent Specs

This directory contains **short design intent documents** for larger features or flows in Second Brain.

They complement the more detailed `.planning/` artifacts by providing:

- The \"why\" and constraints for a feature
- High-level behavior and UX expectations
- Links to relevant knowledge entries in `docs/knowledge/`

Specs should be:

- Small (1–2 pages)
- Opinionated about intent
- Updated when intent changes, not for every implementation detail
