# Phase 10.3: Swift Migration - Settings UI

**Goal**: Implement a preferences window to allow users to configure the Vault Path and select an Ollama Model.

## User Review Required

> [!NOTE]
> This phase transforms the hardcoded values (if any) into user-configurable settings, persisting them in `UserDefaults`.

## Proposed Changes

### Backend Enhancements
#### [MODIFY] [apple/SecondBrain/Sources/SecondBrain/OllamaClient.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/OllamaClient.swift)
- **Method**: `listModels() async throws -> [String]`
- **Implementation**: Fetch `GET /api/tags`, parse JSON, return model names.

### UI Components
#### [NEW] [apple/SecondBrain/Sources/SecondBrain/UI/SettingsView.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/UI/SettingsView.swift)
- **View**: `SettingsView`
- **Features**:
    - **Vault Path**: Text field + "Browse" button (`NSOpenPanel`).
    - **Model Selection**: Picker/Dropdown populated by `OllamaClient.listModels`.
    - **Save**: Persists to `UserDefaults` (using `@AppStorage`).

#### [NEW] [apple/SecondBrain/Sources/SecondBrain/ViewModels/SettingsViewModel.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/ViewModels/SettingsViewModel.swift)
- **Class**: `SettingsViewModel` (`ObservableObject`)
- **State**: `vaultPath`, `selectedModel`, `availableModels`.
- **Logic**: Fetches models on appear, handles file dialog.

### Integration
- **Entry Point**: `SecondBrainApp.swift` handles `Settings` scene (already present, need to link `SettingsView`).
- **MenuBarManager**: Update "Settings..." menu item to open the settings window.
