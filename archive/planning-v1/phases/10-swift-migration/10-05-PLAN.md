# Phase 10.5: Swift Migration - Packaging & Polish

**Goal**: Transform the raw Swift executable into a fully-fledged macOS Application (`.app`), enable auto-launch on login, and improve the AI classification accuracy.

## User Review Required

> [!NOTE]
> This phase moves from running `swift run` to double-clicking `Second Brain.app`.

## Proposed Changes

### 1. App Packaging
#### [Script] `scripts/bundle_app.sh`
- **Action**: Compile release build `swift build -c release`.
- **Structure**: Create `Second Brain.app/Contents/MacOS`, `/Resources`, `/Info.plist`.
- **Icon**: Convert `resources/icon.iconset` to `.icns` and place in bundle.
- **Info.plist**: Define Bundle ID (`com.richardyu.SecondBrain`), `LSUIElement` (true) for Menu Bar app.

### 2. Auto-Launch
#### [modify] [SettingsView.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/UI/SettingsView.swift)
- **Feature**: Add "Start at Login" checkbox.
- **Logic**: manage a `LaunchAgent` plist in `~/Library/LaunchAgents` that points to the `.app` executable.
    - *Note*: `SMAppService` requires code signing/provisioning which might be overkill for local dev. A simple plist write is robust for personal use.

### 3. Prompt Refinement
#### [modify] [NoteProcessor.swift](file:///Users/richardyu/PARA/01_Projects/Personal/apps/second-brain/apple/SecondBrain/Sources/SecondBrain/Services/NoteProcessor.swift)
- **Prompt**: Update the classification prompt to:
    - Include specific definitions of PARA.
    - Ask for a concise title.
    - Force strict JSON format.
