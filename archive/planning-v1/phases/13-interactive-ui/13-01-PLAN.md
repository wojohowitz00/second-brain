# Phase 13.1: Data Layer (GRDB + SQLite)

**Goal**: Establish a persistent data layer so the app can track sync events, note classifications, and enable search across processed notes.

## User Review Required

> [!IMPORTANT]
> This adds a third-party dependency ([GRDB.swift](https://github.com/groue/GRDB.swift)) to Package.swift. GRDB is the most mature Swift SQLite library (14k+ GitHub stars, actively maintained). It adds ~2MB to the final binary.

### 2. Database Manager

#### [NEW] `Sources/SecondBrain/Data/AppDatabase.swift`
- `AppDatabase` class wrapping a `DatabasePool`
- Default location: `~/Library/Application Support/SecondBrain/secondbrain.sqlite`
- Accept custom path for testing (in-memory or temp dir)
- `DatabaseMigrator` with versioned migrations:

**Migration v1** — Core tables:
```sql
CREATE TABLE sync_event (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL DEFAULT (datetime('now')),
    message_count INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'success'
);

CREATE TABLE note (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sync_event_id INTEGER REFERENCES sync_event(id),
    slack_ts TEXT NOT NULL UNIQUE,
    original_text TEXT NOT NULL,
    classified_category TEXT NOT NULL,
    classified_name TEXT,
    classified_filename TEXT NOT NULL,
    classified_title TEXT,
    destination_path TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    corrected_category TEXT,
    corrected_at TEXT
);
```

**Migration v2** — FTS5 search index:
```sql
CREATE VIRTUAL TABLE note_fts USING fts5(
    original_text,
    classified_title,
    classified_name,
    content='note',
    content_rowid='id'
);

-- Triggers to keep FTS in sync
CREATE TRIGGER note_ai AFTER INSERT ON note BEGIN
    INSERT INTO note_fts(rowid, original_text, classified_title, classified_name)
    VALUES (new.id, new.original_text, new.classified_title, new.classified_name);
END;

CREATE TRIGGER note_ad AFTER DELETE ON note BEGIN
    INSERT INTO note_fts(note_fts, rowid, original_text, classified_title, classified_name)
    VALUES('delete', old.id, old.original_text, old.classified_title, old.classified_name);
END;

CREATE TRIGGER note_au AFTER UPDATE ON note BEGIN
    INSERT INTO note_fts(note_fts, rowid, original_text, classified_title, classified_name)
    VALUES('delete', old.id, old.original_text, old.classified_title, old.classified_name);
    INSERT INTO note_fts(rowid, original_text, classified_title, classified_name)
    VALUES (new.id, new.original_text, new.classified_title, new.classified_name);
END;
```

### 3. Data Models

#### [NEW] `Sources/SecondBrain/Data/Models.swift`
- `SyncEvent`: Codable + FetchableRecord + PersistableRecord
- `Note`: Codable + FetchableRecord + PersistableRecord
- Query helpers:
  - `SyncEvent.recentEvents(limit:)` → ordered by timestamp desc
  - `Note.search(query:)` → FTS5 match with snippet extraction
  - `Note.uncorrected()` → notes where `corrected_category IS NULL`
  - `Note.stats()` → count by category
